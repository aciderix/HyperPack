# HyperPack Web

The web and desktop frontend for [HyperPack](../README.md).

## Overview

This is a React + TypeScript application that provides a graphical interface for HyperPack compression. It runs in two modes:

- **Web (WASM)** — Runs entirely in the browser via WebAssembly. No server, no uploads.
- **Desktop (Tauri)** — Native app with full CLI performance via Tauri's shell bridge.

**[▶ Try it online](https://aciderix.github.io/HyperPack/)**

## Tech Stack

- [React](https://react.dev/) 19 + TypeScript
- [Vite](https://vite.dev/) 6
- [Tailwind CSS](https://tailwindcss.com/) 4
- [Tauri](https://v2.tauri.app/) 2 (desktop builds)
- [Emscripten](https://emscripten.org/) (WASM compilation)

## Development

```bash
# Install dependencies
npm install

# Start dev server
npm run dev

# Build for production
npm run build

# Build desktop app
npm run tauri build
```

## Architecture

```
src/
├── App.tsx              # Main component — file picker, compress/decompress UI
├── hooks/
│   └── useHyperPack.ts  # Core hook — manages WASM worker lifecycle
├── lib/
│   └── native.ts        # Tauri bridge — delegates to native binary when available
└── ...

public/
├── worker.js            # Web Worker — loads WASM, handles compress/decompress messages
├── hyperpack.js         # Emscripten glue code (generated by build-wasm.sh)
├── hyperpack.wasm       # Compiled WASM module
└── manifest.json        # PWA manifest
```

### How It Works

1. **User drops a file** → ArrayBuffer transferred to Web Worker
2. **Worker writes to MEMFS** → Emscripten's in-memory filesystem
3. **WASM compresses/decompresses** → Calls `wasm_compress()` or `wasm_decompress()`
4. **Result read from MEMFS** → Transferred back to main thread (zero-copy via Transferable)
5. **Download triggered** → Browser save dialog

In Tauri mode, step 2–4 are replaced by a direct call to the native binary via `Command.create()`.

## Limitations vs CLI

See [docs/WEB_VS_NATIVE.md](../docs/WEB_VS_NATIVE.md) for the full comparison.

Key differences:
- WASM is single-threaded (no `-j N`)
- 64 MB max block size (vs 128 MB native)
- No archive mode in WASM
- ~40–60% slower than native CLI
